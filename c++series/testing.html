In this article, I would like to go through the procedures on how software and hardware testing takes place in Robotics and Self Driving Vehicle Industry. So let us first look into Software Testing. In Robotics software development, three types of tests are prevalent. 1. Unit Testing, 2. Integration Testing, and 3. Regression Testing.
1. Unit Testing
Unit test refers to testing individual functionality or a section of a software System or application (also called Unit) to ensure its design and behavior with the standard expectations. In procedural programming paradigm, a unit can be an entire module, a function or a procedure. On the other hand, in Object Oriented Programming, a unit can be an entire class or a method of the class.
Some of the common unit testing frameworks are gtest, pytest and unittest. In this article we will be discussing gtest and pytest.
Gtest is a testing framework which is developed by google for testing c++ programs. These can be easily integrated with ROS and hence they are very useful for Robotics Applications.
When using gtest you start writing the Assertion statements that checks whether a condition is true.
